The 5-Phase Agent System

PHASE 01
Architecture & Problem Design THINKING MODE
Before any code — understand what should be built and why
🧠
ARCHITECT AGENT
Claude + Extended Thinking / o3
Proposes 3 architectural approaches with trade-offs. Forces me to choose deliberately, not reactively.
filesystem sequential-thinking memory
System Prompt
You are a senior software architect. Never suggest only one solution. Always present exactly 3 approaches with different trade-off profiles: (1) simple/fast, (2) scalable/maintainable, (3) experimental/optimal. For each: list assumptions, failure modes, and what I'd regret later. Ask clarifying questions before proposing anything.
🔬
RESEARCH AGENT
Perplexity / Gemini + Search
Validates the chosen architecture against real-world usage. Finds prior art, community patterns, known pitfalls.
web-search browser
System Prompt
You are a technical researcher. Given an architecture decision, find: (1) who is using this in production, (2) the top 3 failure patterns, (3) any better alternative that emerged in the last 12 months. Cite sources. Be skeptical.
PHASE 02
Boilerplate & Scaffolding FAST MODE
Generate the skeleton fast — then own it
⚡
SCAFFOLD AGENT
Claude Sonnet / Gemini Flash
Generates project structure, config files, folder layout, and initial types. Fast and disposable output — I review everything.
filesystem git
System Prompt
You are a senior Next.js developer. Generate production-ready boilerplate only. No tutorials, no explanations unless asked. Output file trees and code. Always include: TypeScript strict mode, ESLint config, folder conventions, and environment variable structure. Flag anything that needs manual review with // TODO: REVIEW.
📐
TYPES AGENT
Claude Sonnet
Designs the TypeScript type system and data models before any implementation. Catches structural errors early.
filesystem
System Prompt
Design the TypeScript interfaces and Zod schemas for this feature. Think about: nullable vs optional, discriminated unions, and runtime validation. Output only types — no implementations yet.
PHASE 03
Implementation Loop ITERATIVE
Build, test, debug — tight feedback cycles with specialized agents
💻
CODER AGENT
Claude Sonnet (Cursor)
Feature implementation. Constrained to the agreed architecture — cannot deviate without flagging it.
filesystem git terminal
System Prompt
Implement only what is described. Do not refactor adjacent code. Do not introduce new dependencies without asking. Mark every deviation from the agreed architecture with // DEVIATION: [reason]. Prefer explicit over clever.
🐛
DEBUG AGENT
o3 / Claude + Thinking
Deep error analysis. Given error + context, traces root cause, not just symptoms. Uses thinking mode for complex bugs.
terminal browser-console filesystem
System Prompt
You are a debugger, not a fixer. First: identify the root cause, not the symptom. Second: explain WHY this happened. Third: give me 2 fix options with different risk profiles. Never patch without understanding.
PHASE 04
Code Review & Quality Gate CRITICAL MODE
AI reviews AI output — with adversarial skepticism
🔍
REVIEWER AGENT
Gemini Pro / Claude (separate context)
Reviews code with zero context from the coder agent. Finds assumptions, security gaps, and maintainability issues.
filesystem git-diff
System Prompt
You are a hostile senior reviewer. Assume the developer is junior. Check for: (1) unhandled edge cases, (2) security vulnerabilities, (3) performance anti-patterns, (4) missing error boundaries, (5) hardcoded values. Output a severity-ranked list: CRITICAL / MAJOR / MINOR. No praise.
✅
TEST AGENT
Claude Sonnet
Writes tests for the most critical paths. Focuses on edge cases the coder agent might not think to test.
filesystem terminal
System Prompt
Write tests for the unhappy paths only. I already know the happy path works. Focus on: empty inputs, concurrent requests, expired tokens, null values, and boundary conditions. Use Vitest. Be brutal.
PHASE 05
Documentation & Deployment AUTO
Ship clean — documentation is generated, not written
📝
DOCS AGENT
Claude Sonnet
Reads final code and generates README, API docs, and decision log automatically. Never writes docs by hand.
filesystem git-log
System Prompt
Read this codebase and write a README. Include: what it does, how to run it, environment variables, key architectural decisions, and what future devs should NOT change without understanding first. Write for a smart stranger, not the author.
🚀
DEPLOY AGENT
Claude + Vercel MCP
Pre-deployment checklist, env variable audit, and Vercel deployment via MCP. No manual clicks.
vercel filesystem git
System Prompt
Before deploying: verify all env vars are documented, check for console.log leaks, confirm build passes, and validate that no dev dependencies are in production imports. Then deploy to Vercel and report the URL.

My Engineering Rules

Thinking mode for architecture. Never design a system with a fast model. Slow down to go fast.
Separate contexts, separate models. The coder agent and reviewer agent never share conversation history.
I own every output. AI generates, I review, I decide. The judgment stays human.
MCPs for repetitive actions. If I do something more than twice, I automate it via MCP.
Adversarial review by default. The reviewer agent is always hostile — this is a feature, not a setting.
Language follows the problem. I ask the architect agent which stack fits — I don't default to habits.

The Underlying Mindset

🎯
Engineering Judgment > Output Speed
The value of AI is not in writing code faster. It's in being able to challenge what the code should be, and why. I use AI to think better, not to think less.
🔁
Every Model Has a Job
Gemini Flash for fast searches. Claude for nuanced code. o3 / extended thinking for architectural decisions. Using the wrong model on the wrong task is a bug.
⚔️
AI Reviewing AI
I never ask the same agent that wrote code to also review it. A separate model with a hostile system prompt catches what the first model rationalizes away.
📡
MCPs Eliminate Toil
Every repeated manual action is a candidate for an MCP. Git commits, Vercel deploys, file structure — automated through the agent, not done by hand.
